Spring注解

常用注解

注解 作用
@Configuration 申明配置类，类似XML配置文件
@Bean 在配置类中注册的Spring对象。 类似 XML中 <Bean></Bean>
@ComponentScan 扫描注解
@Scope 指定Bean的作用域
@Lazy 懒加载
@Conditional 按照一定的条件进行判断，满足条件则给容器中注册Bean
@Import 导入组件

@ComponentScan

扫描注解，可以有多种方法进行扫描的配置。

  • 扫描指定包下的注解
    @Configuration
@ComponentScan(value = "com.myproject.example.springanno")
public class AnnoConfig {
}
  • 通过TypeFilter指定过滤规则
    @Configuration
@ComponentScan(value = "com.myproject.example.springanno",includeFilters = {
           //  指定具体的注解
        //      @ComponentScan.Filter(type = FilterType.ANNOTATION,classes = {Controller.class}),
        // 指定具体的某个类
       // @ComponentScan.Filter(type = FilterType.ASSIGNABLE_TYPE,classes = {AnnoService.class}),
       // 指定自定义规则
        @ComponentScan.Filter(type = FilterType.CUSTOM,classes = {MyTypeFilter.class}),
})
public class AnnoConfig {
}
    @Filter 定义过滤规则
    excludeFilters= Fileter[] 指定扫描时按照的什么规则排斥哪些组件
    includeFilters= Fileter[] 指定扫描时，只需要包含哪些组件
    指定type = FilterType.CUSTOM 时需要指定自定义实现接口TypeFilter的规则，如下：
public class MyTypeFilter implements TypeFilter {
    @Override
    public boolean match(MetadataReader metadataReader, MetadataReaderFactory metadataReaderFactory) throws IOException {
        // MetadataReader metadataReader : 读取到当前正在扫描的类的信息
        // MetadataReaderFactory metadataReaderFactory : 获取到的其他任何类的信息
        // 获取当前类注解的信息
        AnnotationMetadata annotationMetadata = metadataReader.getAnnotationMetadata();
        // 获取当前正在扫描的类的信息
        ClassMetadata classMetadata = metadataReader.getClassMetadata();
        // 获取当前类资源（类路径）
        Resource resource = metadataReader.getResource();
        return false;
    }
}

public enum FilterType {

	/**
	 * 过滤指定注解，默认是扫描所有注解
	 * @see org.springframework.core.type.filter.AnnotationTypeFilter
	 */
	ANNOTATION,

	/**
	 * 过滤给定类型
	 * @see org.springframework.core.type.filter.AssignableTypeFilter
	 */
	ASSIGNABLE_TYPE,

	/**
	 * 筛选匹配给定AspectJ类型模式表达式的注解
	 * @see org.springframework.core.type.filter.AspectJTypeFilter
	 */
	ASPECTJ,

	/**
	 * 过滤匹配给定正则表达式模式的注解.
	 * @see org.springframework.core.type.filter.RegexPatternTypeFilter
	 */
	REGEX,

	/** 
	 * 使用给定的自定义过滤
	 * {@link org.springframework.core.type.filter.TypeFilter} implementation.
	 */
	CUSTOM

}

@Scope

指定Bean 的作用域

@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface Scope {

	@AliasFor("scopeName")
	String value() default "";

    /**
     *   prototype 原型，每次获取实例，创建一次实例
     *   singleton 单例（默认值）， IOC容器启动会调用方法创建对象放到IOC容器中，以后每次获取就是直接从容器（map.get()）中拿
     *   request 同一个请求创建一个实例，在web环境中使用
     *   session 同一个session创建一个实例，在web 环境中使用
     */
	@AliasFor("value")
	String scopeName() default "";

	ScopedProxyMode proxyMode() default ScopedProxyMode.DEFAULT;

}

@Import

  1. @Import(要导入到容器中的组件)，容器中就会自动注册这个组件，id默认是全名
@Configuration
@Import(Color.class)  // 会将Color类注册成Bean
public class AnnoConfig {
}
class Color{
}
  1. ImportSelector:返回需要导入的组件的全类名数组（SpringBoot自动配置使用的该方法,见EnableAutoConfigurationImportSelector.class 的实现）
@Configuration
@Import(MyAnnoImportSelector.class)  // 会将MyAnnoImportSelector返回的一系列类注册成Bean
public class AnnoConfig {
}
public class MyAnnoImportSelector implements ImportSelector {
    @Override
    public String[] selectImports(AnnotationMetadata importingClassMetadata) {

// 返回值不能为NULL，否则会抛NPE
        return new String[0];
    }
}
  1. ImportBeanDefinitionRegistrar :手动注册bean到容器中
@Configuration
@Import(MyAnnoImportBeanDefinitionRegistrar.class)  // 实现注册Bean的逻辑
public class AnnoConfig {
}
public class MyAnnoImportBeanDefinitionRegistrar implements ImportBeanDefinitionRegistrar {
    @Override
    public void registerBeanDefinitions(AnnotationMetadata importingClassMetadata, BeanDefinitionRegistry registry) {

        //AnnotationMetadata importingClassMetadata ：当前类的注册信息
        // BeanDefinitionRegistry registry ： BeanDefinition注册类，
        // 把所有需要添加到容器中的bean，通过调用BeanDefinitionRegistry.registerBeanDefinition方法手工进行注册
        
        // 通过RootBeanDefinition进行bean 的定义，然后注册
        RootBeanDefinition rootBeanDefinition = new RootBeanDefinition(MyAnno.class);
        registry.registerBeanDefinition("myAnno",rootBeanDefinition);
    }
}

给容器注册组件的四种方式

1. 包扫描 + 组件标注注解

一般是用于自己定义的bean

2. @Bean

一般用于导入第三方包里面的组件

3. @Import

快速导入一个组件

4. @使用Spring 提供的FactoryBean(工厂Bean)

  1. 默认获取的是工厂Bean调用getObject创建的对象
  2. 要想获取工厂Bean本身，需要在id前面加一个& ，如 AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext(AnnoConfig.class); Object beanFactory = context.getBean("&myAnnoFactory");

FactoryBean的使用

  1. 创建一个FactoryBean
    class MyAnnoFactory implements FactoryBean<MyAnno>{

        // 返回一个Myanno 对象，会注册到容器中的

        @Override
        public Class<?> getObjectType() {
            return MyAnno.class;
        }

        // 设置是否单实例bean
        @Override
        public boolean isSingleton() {

org.springframework.context.annotation.AnnotatedBeanDefinitionReader#register

基于Annotation 装载spring bean元信息

private <T> void doRegisterBean(Class<T> beanClass, @Nullable String name,
			@Nullable Class<? extends Annotation>[] qualifiers, @Nullable Supplier<T> supplier,
			@Nullable BeanDefinitionCustomizer[] customizers) {

        // 1. 获取beanClass上的注解元信息，包装的在AnnotatedGenericBeanDefinition中
		AnnotatedGenericBeanDefinition abd = new AnnotatedGenericBeanDefinition(beanClass);
		if (this.conditionEvaluator.shouldSkip(abd.getMetadata())) {
			return;
		}

		abd.setInstanceSupplier(supplier);
		ScopeMetadata scopeMetadata = this.scopeMetadataResolver.resolveScopeMetadata(abd);
		abd.setScope(scopeMetadata.getScopeName()); // 作用域设置
		// beanname自动生成
		String beanName = (name != null ? name : this.beanNameGenerator.generateBeanName(abd, this.registry));
        // 2. 一些特定的注解属性设置 @Lazy、@Primary、@DependsOn、@Role、@Description
		AnnotationConfigUtils.processCommonDefinitionAnnotations(abd);
		if (qualifiers != null) {
			for (Class<? extends Annotation> qualifier : qualifiers) {
				if (Primary.class == qualifier) {
					abd.setPrimary(true);
				}
				else if (Lazy.class == qualifier) {
					abd.setLazyInit(true);
				}
				else {
					abd.addQualifier(new AutowireCandidateQualifier(qualifier));
				}
			}
		}
		if (customizers != null) {
			for (BeanDefinitionCustomizer customizer : customizers) {
				customizer.customize(abd);
			}
		}
		
        // 将的bean元信息包装到BeanDefinitionHolder中
		BeanDefinitionHolder definitionHolder = new BeanDefinitionHolder(abd, beanName);
		definitionHolder = AnnotationConfigUtils.applyScopedProxyMode(scopeMetadata, definitionHolder, this.registry);
		// 注册bean
		BeanDefinitionReaderUtils.registerBeanDefinition(definitionHolder, this.registry);
	}
